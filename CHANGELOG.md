# 更新日志

## [4.8.6] - 2025-01-22

### 🔐 重大功能更新：智能多仓库认证管理系统

#### 🎯 核心功能特性
- **智能多仓库认证管理**: 支持不同SVN仓库的独立认证信息存储和管理
- **项目级精确认证**: 基于完整SVN项目URL进行认证匹配，支持文件夹级权限控制
- **三级认证优先级**: 系统默认认证 → 保存的认证信息 → 用户输入认证
- **自动认证重试机制**: 认证失败时智能提示用户输入新的认证信息

#### 🔧 技术实现亮点
- **安全存储**: 使用VSCode SecretStorage API加密存储认证信息
- **智能URL匹配**: 支持精确匹配、父子路径匹配和降级匹配策略
- **自动认证检测**: 智能识别SVN认证失败错误（E170001等）
- **认证信息继承**: 子项目可自动使用父项目的认证信息

#### 🎨 用户界面优化
- **认证管理面板**: 可视化管理所有保存的认证信息
- **认证输入对话框**: 友好的认证信息输入界面，支持保存选择
- **实时认证验证**: 一键测试已保存认证信息的有效性
- **批量认证管理**: 支持查看、编辑、删除、清除认证信息

#### 🚀 使用体验提升
- **无缝认证体验**: 首次输入后自动保存，后续操作无需重复输入
- **多项目支持**: 完美支持多个SVN仓库的不同认证信息
- **权限分离**: 支持基于项目路径的精细化权限管理
- **智能降级**: 获取完整URL失败时自动使用仓库根URL

#### 📁 新增文件
- `src/svnAuthService.ts` - SVN认证管理核心服务
- `src/svnAuthDialog.ts` - 认证输入和管理界面

#### 🔄 核心修改
- `src/svnService.ts` - 集成认证重试机制到executeSvnCommand
- `src/extension.ts` - 注册认证管理相关命令
- `package.json` - 添加认证配置项和命令定义
- `README.md` - 新增认证管理功能文档

#### ⚙️ 配置选项
```json
{
  "vscode-svn.authenticationConfig.autoSaveCredentials": true,
  "vscode-svn.authenticationConfig.defaultAuthPrompt": true,
  "vscode-svn.authenticationConfig.credentialTimeout": 86400,
  "vscode-svn.authenticationConfig.serverPatterns": {}
}
```

#### 🎛️ 新增命令
- `SVN: 管理认证信息` - 打开认证管理面板
- `SVN: 清除认证信息` - 清除所有保存的认证信息
